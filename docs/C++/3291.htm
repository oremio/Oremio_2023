什么是智能指针？

C++ 标准库中用于解决传统指针内存泄漏问题的一些类。利用变量离开作用域自动调用析构函数的特点，完成了对指针引用块的计数，并在适当的时候自动销毁所指内存，一定程度上避免了内存泄漏的问题。

目前 auto_ptr 已经被弃用，标准库中的智能指针分为三种：shared_ptr，weak_ptr，unique_ptr

 - unique_ptr：保持了一段时间内只有一个指针指向该对象，可以通过 move 操作来在 unique_ptr 之间实现指针所有权的转移

 - shared_ptr：多个 shared_ptr 可以指向同一片内存，采用了引用技术的机制。当最后一个引用被销毁时，自动销毁所指向的内存。

 - weak_ptr：用于解决 shared_ptr 循环引用导致的死锁问题；它指向了由 shared_ptr 管理的内存，但不会影响该 shared_ptr 的管理：这表现为不占用引用计数和不阻碍 shared_ptr 销毁。不可以通过 weak_ptr 直接使用对象，但是可以通过它获得 shared_ptr 实例从而引用对象。

互锁情形：两个类包含了指向对方的智能指针，并创建对象，修改该成员为指向对方的智能指针；这段代码所在的作用域结束后，这两个指针指向的空间不会被释放。
