C++ 的强制类型转换有哪些？

const_cast、static_cast、dynamic_cast、reinterpret_cast

static_cast：可以完成内置数据类型和继承关系中的转换，但不会进行运行时的检查。因此，由父类向子类的转换是不安全的。不能用于无关类型和指针的转换，也不能取消 const 属性，但是可以增加 const 属性。

const_cast：只能用于指针或者引用，可以修改目标的 const 属性（将 const 变为非 const，vice versa）；唯一去除 const 描述符的方法，它仅修改指针或引用本身的 const 属性，而并不会修改其关联对象的 const 属性；修改指向 const 对象的 const 指针为非 const，并使用该指针修改对象的后果是 UB，取决于编译器的实现。

dynamic_cast：可以完成从基类指针到派生类指针的安全转换，转换过程中借助了运行时的安全检查，需要运行时的类型信息（RTTI）。转换指针失败则会返回 nullptr，转换引用失败会抛出 bad_cast；由于转换的过程中需要通过虚表获得类型信息，因此要求基类包含虚函数。

与 RTTI 相关的还有 typeid 运算符，返回 type_info 的引用，该类定义在标准库中。如果表达式的类型包含了虚函数，那么 typeid 在返回类型时会进行动态计算；否则在编译时就可以计算。可以通过 type_info 中的 name 方法获得类型的名称，但是这是由编译器自行生成的，C++ 标准并没有规定其格式。

reinterpret_cast：不做任何安全检查，和 C 风格强制转换有完全一致的能力；仅将目标数据重新解释，而不会对数据做任何修改。
