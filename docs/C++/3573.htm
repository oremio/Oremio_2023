在采用虚继承的时候，直接由最低层次的派生类构造函数直接负责虚基类的构造。

如果不加virtual的话，在构造函数的顺序中，每个类只负责自己的直接基类的初始化，所以还是可以生成对象的。

加上了virtual之后，C直接负责Base类的构造，但是Base类的构造函数和析构函数都是private，C无法访问，所以不能生成对象。

代码示例：

#include <iostream>

class A
{
public:
    friend class B;
private:
    A() {}
};

class B : virtual public A
{
};

// class C : public B {};

int main()
{
    B b;
    return 0;
}

  1. 虚继承：类 B 通过虚继承的方式继承自类 A。虚继承用于解决多重继承中的菱形继承问题，确保共享基类（这里是 A）在继承层次中只有一个实例。在虚继承中，虚基类的构造函数是由最派生类（这里是类 C 或者 C 的派生类）负责调用。

  2. 类 C 的构造：当尝试构造类 C 的对象时，虚基类 A 的构造函数需要被调用。根据C++的规则，即使类 B 是类 A 的友元，这种友元关系不会传递给类 B 的派生类（这里是类 C）。因此，类 C 无法访问类 A 的私有构造函数。
