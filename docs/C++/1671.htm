在C++中，stdin和cin都是用于从标准输入设备（通常是键盘）读取输入数据的方式，但它们有几个重要的区别：

  1. 类型差异：

    • cin是C++中的标准输入流，属于istream类的对象，使用C++标准库提供的输入运算符（>>）来读取输入。
      •
    • stdin是C语言中的标准输入流，是一个FILE*类型的指针，使用C标准库函数（如fgets()、fscanf()等）来读取输入。

  2. 面向对象：

    • cin是C++中的面向对象输入流，支持面向对象的操作，如输入运算符重载等。
    • stdin是C语言中的基于文件流的输入方式，更接近底层，不支持面向对象的操作。

  3. 异常处理：

    • cin在读取数据时，可以进行类型检查和错误处理，例如输入数据类型不匹配时会触发输入流的错误状态，可以通过异常处理或者流状态检查来处理错误。
    • stdin的输入操作相对较为底层，不会进行类型检查和错误处理，需要手动编写代码来处理输入错误。

总的来说，cin是C++中更加高级、方便和安全的输入方式，而stdin则更加底层，适用于C语言和需要更多控制的场景。

0x3f 是一个在编程中常见的十六进制值。在特定情况下，我们会用它来表示“无穷大”或其他特殊情况。

这个值通常用于初始化变量，以代表一个极大的数，类似于正无穷大。计算机的存储空间是有限的，无法直接存储无限大的数，因此我们使用 0x3f3f3f3f 来表示这个概念。

例如，当我们需要将一个数组的元素初始化为一个极大值时，可以使用 memset() 函数。下面是一些示例：

  1. 一维数组：将所有元素初始化为 0x3f3f3f3f：

    int a[10];
memset(a, 0x3f3f3f3f, sizeof(a));

    这样，数组 a 中的每个元素都被设置为一个很大的数，可以代表正无穷大。

  2. 二维数组：同样，我们可以对二维数组进行类似的操作：

    int a[10][10];
memset(a, 0x3f3f3f3f, sizeof(a));

    这样，二维数组 a 中的每个元素都被初始化为 0x3f3f3f3f。

总之，0x3f3f3f3f 是一个常用的值，用于表示无穷大或其他特殊情况，特别在算法中经常会遇到。

size_t主要用于计数（例如数组下标），如sizeof函数返回值类型即为size_t。在不同位的机器中所占的位数也不同，size_t是无符号数，ssize_t是有符号数。

  • 在32位机器中定义为：typedef unsigned int size_t; （4个字节）
  • 在64位机器中定义为：typedef unsigned long size_t;（8个字节）

由于size_t是无符号数，因此，当变量有可能为负数时，必须使用ssize_t。

assert()依赖于NDEBUG 宏

NDEBUG这个宏是C/C++标准规定的，所有编译器都有对它的支持。

1. 调试(Debug)模式编译时，编译器不会定义NDEBUG，所以assert()宏起作用

2. 发行(Release)模式编译时，编译器自动定义宏NDEBUG，使assert不起作用

如果要强制使得assert()生效或者使得assert()不生效，只要手动 #define NDEBUG 或者 #undef NDEBUG即可。

 

static_assert (bool_constexpr, message)

其中两个参数解释如下： 

(1)      bool_constexpr:   编译期常量表达式，可转换为bool 类型

(2)      message: 字符串字面量 ，是断言失败时显示的警告信息。自C++17起，message是可选的

 举例：下面的语句能够确保该程序在32位的平台上编译进行。如果该程序在64位平台上编译，就会报错

static_assert(sizeof(void *) == 4, "64-bit code generation is not supported.");

 

static_assert的用途：常用在模版编程中 ，对写库的作者用处大
