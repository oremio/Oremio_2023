假设我们有一个表示学生信息的类Student，其中包含私有成员name和age。现在我们编写一个成员函数getName()，目的是返回学生的姓名。我们来比较一下两种不同的实现方式：

错误的实现：

#include <iostream>
#include <string>
​
class Student {
private:
    std::string name;
    int age;
​
public:
    // 错误的实现：返回非常量引用指向非常量成员
    std::string& getName() {
        return name;
    }
​
    // 其他成员函数...
};

正确的实现：

#include <iostream>
#include <string>
​
class Student {
private:
    std::string name;
    int age;
​
public:
    // 正确的实现：返回常量引用指向常量成员
    const std::string& getName() const {
        return name;
    }
​
    // 其他成员函数...
};

在错误的实现中，getName()函数返回了一个非常量引用，指向了name成员变量。这样的设计可能会导致外部代码对name的修改，破坏了类的封装性和数据的一致性。

而在正确的实现中，getName()函数返回了一个常量引用，指向了name成员变量。这样的设计可以确保外部代码无法修改name，同时保护了类的封装性和数据的一致性。

因此，根据条款的建议，应该尽量避免返回非常量指针或引用，并且指向非常量成员的成员函数，而应该返回常量指针或引用，指向常量成员。
