Topic #3,586: 千万小心地使用智能指针的get (top)

void process(shared_ptr<int> ptr)
{
    cout << ptr.use_count() << endl;
}

int main() {
    shared_ptr<int> p(new int(32));
    process(shared_ptr<int>(p.get()));
    cout << p.use_count() << endl;
    return 0;
}

问题分析

  1. 创建原始 shared_ptr：

    • shared_ptr<int> p(new int(32)); 这行代码创建了一个智能指针 p，负责管理一个动态分配的 int 对象（值为 32）。这里 p 是对该内存块的唯一持有者。

  2. 传递给 process 函数：

    • process(shared_ptr<int>(p.get())); 这行代码尝试通过 p.get() 获取原始指针并用它来构造一个新的 shared_ptr 对象。然而，这是非常危险的操作，因为 p.get() 仅返回指向管理对象的指针，而不传递所有权。这意味着新创建的 shared_ptr（在 process 的参数中）和原始的 p 相互独立，它们都认为自己拥有对该对象的独立所有权。

  3. 内存管理冲突：

    • 由于 process 函数的 ptr 和 main 中的 p 都认为自己拥有对同一个 int 对象的所有权，因此在它们的生命周期结束时，都会尝试删除同一个内存地址。这将导致 delete 同一个内存地址两次，从而引起程序崩溃或其他未定义行为。

后果

  • 双重删除：当 process 函数结束后，局部 shared_ptr ptr 的析构函数会调用 delete 释放内存。然后，当 main 函数结束时，p 也会尝试删除同一个内存块。这种双重删除是未定义行为，最常见的后果是程序崩溃。

因此，请千万小心地使用智能指针的get。

要好好理解这句话：异常中断了程序的正常流程。异常发生时，调用者请求的一部分计算可能已经完成了，另一部分则尚未完成。通常情况下，略过部分程序意味着某些对象处理到一半就戛然而止，从而导致对象处于无效或未完成的状态，或者资源没有正常释放，等等。那些在异常发生期间正确执行了“清理”工作的程序被称作异常安全(exception safe)的代码。

我们必须时刻清楚异常何时发生，异常发生后程序应如何确保对象有效、资源无泄漏、程序处于合理状态，等等。
